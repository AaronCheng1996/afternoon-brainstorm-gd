# 07 爬塔模式規格（Python endless/）＋ 新功能建議

> Python 版已內建完整爬塔（endless）模式，Godot 直接移植它就是「爬塔模式」。
> 出處：`endless/run_state.py`、`scaling.py`、`content.py`、`effects.py`、`run_loop.py`、
> `endless_ai.py`、`battle_builder.py`、`endless_save.py` 與各 screen。

## 1. 模式總流程

```
爬塔選單（繼續 / 新開局 / 最佳紀錄）
  └─ 每層循環：
     樓層預覽（顯示層數/種類/敵方效果/我方效果）→ 對戰（mode=campaign + EndlessAIController）
       ├─ 敗 → 結算畫面（記錄最高層）→ 回選單
       └─ 勝 → temp_deck 消耗結算 → 發金幣 → 獎勵三選一（卡/遺物/跳過+10金）
              → 每 5 層進商店 → 下一層
     事件層：不戰鬥，進事件畫面（噴泉/賭徒/祭壇）→ 直接下一層
```

玩家初始牌組 = WHITE_DECK；`run.deck`（永久）+ `run.temp_deck`（獎勵得來的卡，**死亡或使用即消耗**）。

## 2. Run 狀態 schema（存檔 `user://endless_save.json` 需相容）

```json
run: {
  "seed": int, "floor": int, "phase": "battle|reward|shop",
  "deck": [card_id], "temp_deck": [card_id],
  "coins": int, "relics": [id], "curses": [id],
  "perm_buffs": {unit_hp_plus, unit_damage_plus, job_damage_plus:{job:int}, ...},
  "armed_consumables": [id], "pending": {...當前獎勵/商店快照},
  "shop_counters": {remove_card, reroll, dmg_all, hp_all},
  "floor_spec": {...} | null, "prev_kind": str
}
state 外層另有 best_floor 紀錄。
```

**決定性 RNG**：樓層內容 = `Random(seed×1000003 + floor×97 + salt)`；
salt：0=floor_spec、1=獎勵選項、3+reroll=商店。→ 同 seed 重進遊戲內容不變（防 SL 刷獎勵）。

## 3. 樓層種類與縮放（scaling.py）

- 種類：`floor % 10 == 0` → **boss**；floor≥4 且 15% 且上層非事件 → **event**；
  floor≥4 且 35% → **elite**；floor≥6 且 55% → **mutation**；否則 normal。（roll 一次依序判定）
- AI 牌組：35% 抽「成套牌組表」（tier0：白/紅；tier1：四色；tier2：混色+新色 9+3；
  tier3：BOSS/META/特化組），否則按模板生成（balanced/aggro/fortress/harvest/snipers ×
  分層色盤 TIER_PALETTES）。層數決定 tier：≤3→0、≤9→1、≤14→2、≥15→3。
- AI 策略：≤3 層 white；單色套牌用該色策略；混色或 ≥15 層用 boss；**boss 層用 claude**
  （beam_width=min(10, 4+2×(floor/10))、depth_cap=min(8, 2+2×(floor/10))）；
  非 boss 層 attack_min_score = max(6, 12 − floor×0.4)（越深越兇）。
- AI 數值縮放（ai_effects）：`unit_hp_plus = floor//8 (+1 若 elite/boss)`；
  `hand_plus = min(3, floor//10) (+1 若 elite)`；floor≥12 → 運氣+min(25,floor)（綠/混色牌組）、
  每 max(4, 8−floor//10) 回合免費治療；floor≥15 → 每 3 回合免費移動。
- 金幣獎勵：`12 + 2×floor`，elite ×2、boss ×3（再乘遺物/詛咒的 coin_mult）。

## 4. 突變（mutation，該層全域規則）

sharp_world 全體+1 ATK｜tough_world 全體+2 HP｜sudden_death 8 分勝｜marathon 12 分勝｜
open_war 雙方起手+2。`both_` 前綴 = 雙方套用；`win_threshold` 直接覆寫核心。

## 5. 遺物 / 詛咒 / 消耗品（content.py）

**遺物（永久，擊敗 boss 必掉一選項；floor≥3 起 25% 出現在獎勵）**：
war_drum(+1ATK)、iron_plate(+2HP)、lucky_charm(+15 運)、supply_cache(+1 起手)、
field_medic(每 4 回合免費治療)、swift_boots(每 4 回合免費移動)、war_chest(金幣×1.5)、
merchant_pass(商店 8 折)。

**詛咒（商店購買，強力但有代價）**：berserker_brew(+2ATK / 起手−1)、
golden_greed(金幣×2 / 敵+2HP)、glass_cannon(+2ATK / −2HP)、heavy_purse(商店半價 / 敵+1 起手+10 運)。

**消耗品（下一戰有效）**：battle_ration(每 3 回合治療)、adrenaline(每 3 回合移動)、scout_report(起手+2)。

**效果合併規則（run_state.merged_player_effects）**：加法鍵直加；週期鍵取**最小 n**（較頻繁者優先）；
job_ 鍵按職業字典累加；coin_mult / shop_discount 相乘。
玩家效果來源 = perm_buffs + 遺物 + 詛咒 player_effects + 消耗品；
AI 追加效果 = 樓層 ai_effects + 詛咒 ai_effects + mutation 的 both_。

## 6. 商店（每 5 層，reward 之後）

固定兩項永久強化：全隊+1ATK（90 + 30×已購次數）、全隊+2HP（70 + 20×已購次數）；
隨機職業+1ATK（45）；隨機 2 種消耗品；未擁有詛咒隨機 1 個（40）；隨機 3 張卡
（魔法 25；TANK/HF 35、其他 30；強卡表 +10、弱卡表 −10）。全部價格乘 shop_discount。
卡牌上限：單位 ≤3 份、魔法 ≤4 份（deck+temp_deck 合計）。

## 7. EndlessAIController 特有行為（endless_ai.py）

繼承 AIController：雙方單位 buff 維護（新單位上場補 hp/dmg，含 job_ 鍵）、
雙方 one-shot（運氣）與起手調整、雙方週期 buff（玩家的也由它施加）、
**玩家消耗追蹤 p1_consumption**：記錄玩家本場打出的每種卡與死亡數、魔法使用數
（MOVEO 因回合結束自然消失不算使用），戰後 `settle_temp_deck` 從 temp_deck 扣掉。

## 8. 決定性測試（P5-1 驗收用）

固定 seed=12345：floor1 必為 normal + tier0 牌組；floor10 必為 boss、claude 策略、
coin_reward=(12+20)×3=96；同 seed 呼叫兩次 `floor_spec` 結果完全相同；
`generate_shop_stock(reroll=0/1)` 結果不同但各自可重現。
（以上由執行者跑 Python 端實值後寫死進測試，確保兩引擎生成邏輯一致——
注意 D4 決策：跨引擎 RNG 序列不同，所以此測試改為「**Godot 內部**同 seed 可重現」＋
「規則面欄位（kind 判定門檻、縮放公式、價格公式）與 Python 公式一致」的單元測試。）

## 9. 新功能建議（供專案擁有者挑選，P5-5 實作 1–2 個）

**低成本（直接加在現有結構上）**：
1. **分岔路線圖**：每層改為 2–3 個節點供選（戰鬥/精英/事件/商店），floor_spec 已含 kind，
   只需一個節點圖生成器 + 地圖畫面。體驗提升最大的一項。
2. **更多事件**：Python 只有 3 個事件（噴泉/賭徒/祭壇）。事件本質是「選項 → 改 run 狀態」，
   加 5–8 個（例：僱傭兵—付金幣得強卡、鍛爐—犧牲 HP 上限換全隊 ATK、占卜—預看未來 3 層）。
3. **牌組移除服務**：shop_counters 已預留 `remove_card`/`reroll` 欄位但商店沒賣——補上
   「移除一張卡（價格遞增）」與「重擲商品」兩個商品。
4. **開局遺物三選一**：新開局先選一個初始遺物，增加 build 差異。

**中成本**：
5. **上升難度（Ascension）**：通關（如到 20 層）後解鎖 A1、A2…，每級疊一條全域負面
   （敵 hand_plus+1、金幣 −20%…），複用 ai_effects 合併機制。
6. **精英詞綴**：elite 層隨機掛 1 條特殊規則（敵方 SP 免疫斬殺、我方每回合 −1 運）——
   結構同 mutation，做一張新表即可。
7. **每日挑戰**：以日期為 seed + 固定 mutation 組合，記排行榜（本機）。
8. **卡牌強化**：商店/事件可把單卡永久 +1/+1（perm_buffs 已支援 job 級，擴充到 instance 級需在
   deck 裡存 `{card_id, plus}` 結構——建議 deck 項目改物件時一併做）。

**高成本（另立專案再議）**：
9. 跨局 meta 進度（爬塔貨幣解鎖新色起始牌組）；10. 成就系統；11. 雲端/好友排行。

**演出面建議（配合目標 2）**：樓層預覽加 boss 立繪佔位與入場橫幅；遺物/詛咒在戰鬥 HUD 顯示圖示列；
獎勵翻牌動畫；爬塔失敗結算的 run 統計回顧（每層耗時/擊殺數——statistics 已有資料）。
