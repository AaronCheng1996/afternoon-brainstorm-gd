# 09 程式碼導覽（code review 入口）

> 目的：快速掌握每個目錄、每個檔案的職責與彼此關係。
> 行為規格見 `01`/`02`，AI 規格見 `03`，架構決策見 `00`/`04`，連線規格見 `10`，
> 各檔完成時的實作備註見 `歸檔_已完成任務.md`。
> 基準：2026-07-16（Phase 8–11 收官、Phase 12 連線動工前），**1817 個 headless 測試全綠**。
> 行數為當時概數，只是「review 這檔要花多少力氣」的參考。

## 1. 一張圖看懂資料流（Sim/View 分離，決策 D1）

```
玩家輸入（滑鼠/鍵盤）─┐
CPU AI（AIController）─┤   ←「action 來源」可替換：本機玩家 / AI / 回放檔 /（Phase 12）遠端玩家
回放（ReplayLog）───────┘
   │  組出 GameAction（純資料）
   ▼
GameCore.dispatch(action)          ←── script/core：純邏輯，RefCounted，零 Node 依賴
   ├─ 改狀態：PieceState / PlayerState / BoardState / Statistics
   ├─ 跑規則：TurnEngine（回合）、Combat（傷害管線）、能力系統（ability/）
   └─ 累積 GameEvent（含動畫延遲 delay）到內部佇列
   │
   ▼
場景呼叫 core.drain_events() 取走事件
   │
   ▼
CombatScheduler 依 delay 排時間軸播動畫（PieceView / Projectile / 飄字 / 擊殺鏡頭震動）
   │  is_busy()==true 期間鎖輸入（AI 與回放也等它）
   ▼
動畫播完 → 場景以 core 最終狀態「重建棋盤」重新同步 → 解鎖輸入
```

**鐵律**：`script/core` 下任何檔案不得 `extends Node`、呼叫 `get_tree()`、`load("res://scenes...")`。
`tests/test_integration.gd` 第 4 部分用靜態掃描守住這條線。
好處：headless 可測、AI 可整局模擬、回放（P11-2 已實現）與連線（Phase 12）直接搭在事件模型上。

**表現層鐵律**：瞬時模式（動畫關）必須與逐格播放**結果完全一致**且**不生成任何特效節點**
——多個測試守著這條（`test_piece_view`/`test_combat_scheduler` 的 instant 不變性案例）。

## 11. 建議讀碼順序（review 路線）

1. **資料模型**：`game_config` → `piece_state` → `board_state`/`player_state` →（`statistics`、`rng_service`）
2. **核心流程**：`game_core`（`dispatch` 分派表）→ `turn_engine` → `combat`（十步管線，對照 01 §6）
3. **能力系統**：`trigger` → `ability_component`（沉默/附魔）→ `cards/white.gd`（最簡）→ 挑一色深讀（推薦 `fuchsia.gd`）
4. **事件→動畫**：`game_event` → `combat_scheduler` → `piece_view` → `board_view`（座標系）
5. **AI**：`ai_query` → `ai_evaluator` → `ai_strategy` → `ai_controller`（對照 03）
6. **場景薄殼**：`battle.gd`（輸入→dispatch→重建；AI/回放/計時接線都在這）→ `draft.gd` → `main_menu`/`end_game` → `encyclopedia`
7. **回放與（未來）連線**：`replay_log`（`simulate()` 是理解決定性的最佳入口）→ `10_連線版本.md`
8. **測試**：`run_tests.gd` → `test_integration.gd` → `test_replay.gd`

## 12. Review 時可留意的慣例與陷阱

- **記憶體**：core 全 RefCounted，Godot 無循環 GC——自我/循環參考處用 WeakRef
  （`ability_component.source`、shadow `linker`）或事後 `hit_cards.clear()`（自傷情境）。
  測試 runner 報 orphan；洩漏基準 39/78，新增結構要比照辦理。
- **決定性**：所有規則隨機走 `RngService`；UI 範圍預覽與 AI 查詢**刻意不呼叫 detection**
  （取同距候選集合）以免消耗 rng；鏡頭震動等純演出用全域 randf（不進 RngService）。
- **瞬時模式等價**：任何新演出必須在 instant 下「零特效節點、同最終狀態」，並補不變性測試。
- **statuses/counters**：棋子布林狀態統一走 `statuses` 字典＋API，不加散裝 bool。
- **常數雙源**：`game_config.gd` 鏡像 Python 硬編碼值（改值兩邊同步）；其餘讀 `Balance`。
  AI 常數在 `campaign_setting.json`（`Balance.ai()`）。
- **`%` 綁定模式**：場景腳本用 `_bind_nodes()`＋`%Name`（headless 下 `_ready` 不觸發）；新場景照此。
- **fx_layer 生命週期**：粒子/殘影掛外部 fx 層，發射者 `queue_free` 後特效播完自清；
  fx_layer 為 null 時退回自身（headless/獨立測試安全）。
- **action 來源可插拔**：人類輸入、AIController、ReplayLog、（Phase 12）遠端玩家都只是
  「組 GameAction 餵 `_do`」——新增來源時不要繞過 dispatch。
- **註解慣例**：每檔頂部說明翻譯來源與規格出處章節，繁體中文。
