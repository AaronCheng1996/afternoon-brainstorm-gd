## 方向宣告（使用者決策）

- **D16（2026-07-11）美術與 UI 不再對齊 Python（pygame）版**：畫面、佈局、操作、演出以 Godot 版
  自身的易用性與觀感優先，自由設計。
- **平衡數值與規則行為仍以 Python 版為唯一基準**：`data/balance/` JSON 單向同步機制不變（J），
  規則核心（`script/core`）的行為測試仍以 Python 為準；**任何表現層改動不得改變對局結果**（全套件必須綠）。
- **D18（2026-07-16）連線版＝專用伺服器＋房間制**：使用者提供一台主機跑 dedicated server；
  玩家開房間邀請人加入，房間可公開或上鎖；支援旁觀。規格見 `10_連線版本.md`（N）。
- **D19（2026-07-16；2026-07-18 修訂）手牌為公開資訊**：雙方手牌對所有參與者（雙方玩家＋旁觀者）可見，
  **本機版也一樣**。隱藏資訊＝牌庫抽序與未來隨機（seed）。**（2026-07-18 修訂）seed 僅在「進行中的
  線上對戰」期間對客端隱藏**（防實時模擬未來）；**線上對局結束後 seed 公開**（可下發供回放下載，P12-18），
  **單機/爬塔模式 seed 本即公開**。連線進行中的快照/事件仍不含 seed（P12-4 不變）。
- **D20（2026-07-19）連線對局行動結果須即時呈現**：任何玩家行動成功後，雙方玩家與旁觀者的
  盤面/手牌/資源顯示須**立即更新，不得等到回合交接才一次同步**。P12-12 備註「校正快照僅回合
  交接下發…屬 §4 協定設計、非缺陷」的判定據此修訂（實機測試認定為缺陷）；伺服器權威與
  單一公開快照（D18/D19）不變。實作＝P12-19。
- **D21（2026-07-19）對戰畫面手牌佈局定案**：棋盤置於畫面中央、雙方手牌固定於**左右兩欄**，
  **任何模式皆不做換手互換**（取代 P12-2 的「換手兩列互換」設計）。有主視角時（單人 vs AI、
  連線）主視角手牌恆**左欄**；本機雙人＝P1（先手）左欄、P2 右欄；旁觀/回放無主視角，比照
  P1 左／P2 右。實作＝P12-20。
