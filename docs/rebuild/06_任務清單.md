## 方向宣告（使用者決策）

- **D16（2026-07-11）美術與 UI 不再對齊 Python（pygame）版**：畫面、佈局、操作、演出以 Godot 版
  自身的易用性與觀感優先，自由設計。
- **平衡數值與規則行為仍以 Python 版為唯一基準**：`data/balance/` JSON 單向同步機制不變（J），
  規則核心（`script/core`）的行為測試仍以 Python 為準；**任何表現層改動不得改變對局結果**（全套件必須綠）。
- **D18（2026-07-16）連線版＝專用伺服器＋房間制**：使用者提供一台主機跑 dedicated server；
  玩家開房間邀請人加入，房間可公開或上鎖；支援旁觀。規格見 `10_連線版本.md`（N）。
- **D19（2026-07-16）手牌為公開資訊**：雙方手牌對所有參與者（雙方玩家＋旁觀者）可見，
  **本機版也一樣**；唯一隱藏資訊＝牌庫抽序與未來隨機（seed 只存在權威端）。

- [ ] **P12-4 公開快照序列化**
  - 目標：`GameCore → Dictionary` 公開快照（盤面棋子全欄位、**雙方手牌明細（D19）**、資源/

    `battle` 端「從快照重建盤面」（把 `_rebuild_board` 的資料源一般化為「core 或快照」）。
  - 驗收：headless：對局中途快照→重建，公開狀態逐欄位一致；斷言快照序列化後**不含**

- [ ] **P12-5 伺服器程序與房間管理**
  - 目標：headless 常駐入口（`script/net/server_main.gd`，`--headless -s` 可跑）＋
    `RoomManager`：大廳（房間列表查詢）、建房（名稱、公開/上鎖＋密碼、觀戰開關、房碼）、
    加入（房碼或列表；上鎖驗密碼）、離開/解散、座位（P1/P2）與就緒、多房間並行；
    房間生命週期 等待→BP→對戰→終局→重開或解散。每房一顆權威 GameCore（對戰起）。
  - 驗收：headless 多 peer：建房→列表可見（上鎖房標記）→密碼錯誤被拒→兩人入座就緒→
    滿房第三人轉旁觀或被拒（依觀戰開關）→離開解散。（N §5）
- [ ] **P12-6 連線對戰核心（server 權威）**
  - 目標：client 組 `GameAction` → server `dispatch`（合法性/回合閘）→ 廣播 `GameEvent` 流
    ＋回合交接/關鍵點廣播公開快照（單一份，全房間同一視角，D19）；client 照既有
    事件→`CombatScheduler` 播動畫、非本回合鎖輸入；回合計時 server 權威（逾時 server 端 end_turn）。
  - 驗收：headless loopback 兩 client 打完整局：**兩端最終分數/回合/統計 export 與 server
    完全一致**；非當前玩家的 action 被 server 拒絕；計時逾時由 server 定奪。（N §6）
- [ ] **P12-7 大廳與房間 UI**
  - 目標：主選單「線上對戰」→ 伺服器位址/埠設定（存 settings，預設記住上次）→ 大廳
    （公開房列表＋鎖圖示＋人數/觀戰數、重新整理、輸入房碼加入）、建房面板（名稱/公開/密碼/
    觀戰開關）、房內畫面（雙方座位與就緒、延遲顯示、房碼供邀請、開始）。版本閘拒絕給明確訊息。
  - 驗收：headless node tree＋面板開關/填充/欄位驗證；連線流程【人工】跨機測。
- [ ] **P12-8 連線選秀（BP）**
  - 目標：房間流程串 BP→對戰：`DraftAction` 經 server 權威同步（回合閘/上限規則同本機），
    非編輯方鎖定顯示「對方選牌中」；選秀計時逾時由 **server** 執行 `auto_fill_and_advance` 並廣播；
    完成後 server 建 GameCore 發首份快照進對戰。
  - 驗收：headless 兩 client 走完三階段，雙方牌組兩端一致；逾時補牌只由 server 產生。（N §6）
- [ ] **P12-9 旁觀系統**
  - 目標：以旁觀者身分加入（上鎖房同樣驗密碼；受觀戰開關控制）；收**同一份公開快照＋事件流**
    （＝看得到雙方手牌、看不到牌庫序，D19）；中途加入＝快照＋續播；旁觀者 UI 全唯讀
    （無行動列/手牌不可點）、顯示「旁觀中」與觀戰人數；終局可看統計。
  - 驗收：headless：對局中途第三 peer 以旁觀加入→重建狀態與 server 一致→後續事件同步；
    旁觀者送 action 被拒。（N §7）
- [ ] **P12-10 斷線重連**
  - 目標：席位 token（入座時發、client 存）；玩家斷線→席位保留 N 秒（可設）、對手畫面顯示
    等待；帶 token 重連→逐出舊連線、收復席位、補送快照＋續播；逾時未回→對手可判勝或解散。
    旁觀者斷線直接移除。
  - 驗收：headless：client 斷線→帶 token 重連→狀態一致可續玩；錯 token 被拒；逾時判定。（N §8）
- [ ] **P12-11 部署與穩定性（收官）**
