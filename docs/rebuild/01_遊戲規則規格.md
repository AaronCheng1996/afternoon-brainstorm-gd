# 01 遊戲規則規格（以 Python 4.3.0.0 為準）

> 本文是 Godot 核心邏輯的**唯一規則基準**。每條規則都附 Python 出處，執行者可回頭查證。
> 卡牌個別能力在 `02_卡牌能力總表.md`；本文只講引擎級規則。

## 1. 基本常數

| 項目 | 值 | 出處 |
|---|---|---|
| 棋盤 | 4 × 4（座標 (0,0)–(3,3)） | `config/setting.json` `board_size`、`core/board_config.py` |
| 勝利條件 | 任一方累計分數絕對值 ≥ `win_threshold`（預設 10；爬塔突變可改 8/12） | `game_state.py`、`battling_dispatcher.py` end_turn |
| 分數符號 | player1 得分 → `score` **減**；player2 得分 → `score` **加**。`score<0` 時 player1 領先 | `cards/base.py` `settle()` |
| 牌組 | 12 張；同名單位 ≤ 2、同名魔法（CUBES/MOVE/HEAL）≤ 3 | `core/draft_dispatcher.py` |
| 起手 | 雙方各抽 3 張；**player1 額外 +1 攻擊次數**（先手補償） | `core/player.py` `initialize()` |
| 抽牌上限 | 無手牌上限 | `player.py`（無檢查） |
| 治療魔法量 | +6 HP | `player.py` `heal_card()` 寫死 |
| CUBES | 一張給 2 次放置 CUBE 的機會 | `player.py` `play_card()` |
| 藍球換抽牌 | 3 顆 token 換 1 抽 | `config/setting.json` `how_many_token_to_draw_a_card` |
| 計時 | `timer`（正計時，超時無事）或 `countdown`（600 秒倒數，歸零判輸） | `setting.json` `countdown_time`、`player.py` |
| 運氣初始值 | 雙方 50（%） | `game_state.py` `players_luck` |
| 金幣上限（Cyan） | 50 | `card_cyan.py` `get_coins()` |

對應座標系：Godot 版一律使用 `Vector2i(0..3, 0..3)`，不要沿用舊 Godot 版的 +2 偏移。

## 2. 資料模型（GameState 必備欄位）

出處 `core/game_state.py`。Godot 版 `GameCore` 需具備等價欄位：

```
score:int, turn_number:int, win_threshold:int
players_luck:{p1,p2,neutral:int}      # 運氣（Green）
players_token:{...}                    # 藍球（Blue）
players_totem:{...}                    # 圖騰刻印（DarkGreen）
players_coin:{p1,p2:int}               # 金幣（Cyan）
card_to_draw:{p1,p2:int}               # 待抽張數（逐幀/逐步消化）
skip_turn_draw:{p1,p2:bool}            # 跳過下次回合抽牌（Brown ASS）
number_of_attacks / movings / cubes / heals : {p1,p2:int}
board: 4x4 每格 occupy:bool
player1/player2: deck, hand, draw_pile, discard_pile, on_board, revealed_deck
neutral.on_board                       # CUBE / LUCKYBLOCK（Shadow 掛在本體上，不在此）
pending_attacks: 佇列（防遞迴，見 §7）
pending_combat_events: 動畫事件輸出（見 §10）
rng_seed / rng                         # 一局一顆種子
```

`revealed_deck`：牌組前 6 張 + 之後每抽到一張新名字就加入（AI 的 claude 策略與 LAN 遮蔽用）。

## 3. 回合流程（權威順序）

```
【開局】
p1.initialize: discard_pile = deck 複本 → 抽3張 → number_of_attacks[p1] += 1
p2.initialize: discard_pile = deck 複本 → 抽3張
（注意：第一回合「不執行」p1 的 turn_start，所以 p1 首回合只有 3 張手牌、1 次攻擊）

【誰的回合】 turn_number 偶數 = player1，奇數 = player2

【turn_start(玩家)】                         # player.py
  若 skip_turn_draw[我] → 清為 False、不抽牌；否則抽 1 張
  number_of_attacks[我] += 1
  對我方每個場上棋子：ROUNDS_SURVIVED 統計 +1 → card.refresh()
      refresh(): moving = False → on_refresh() 能力鉤子

【回合中】玩家任意次執行行動（見 §4），受各計數器限制

【end_turn】                                  # battling_dispatcher.py + player.py
  turn_number += 1
  我方 turn_end:
    selected 清除；手牌移除所有 "MOVEO"
    number_of_cubes/movings/heals[我] 歸零（未用完不保留）
    我方每個場上棋子 settle():
        moving = False
        統計 SCORED += on_settle(False)        # 先查詢不清狀態
        分數 += / -= on_settle(True)           # 這次才清 numbness/anger
  記錄 score_history
  若 |score| >= win_threshold → 勝負judge（score<0 → player1 勝）
  對方 turn_start(...)

【每幀 / 每邏輯步 logic_update】               # player.py logic_update + neutral.update
  回收死亡棋子：health<=0 且 can_be_killed() → die() 鉤子 → 進 owner 的 discard_pile
      （neutral 棋子直接消失不進牌庫）→ 釋放格子 occupy
  card_to_draw[我] > 0 → 每步消化 1 張抽牌
  計時器更新
```

**numbness（麻痺/入場暈）規則**——最容易做錯，務必照做：

- 新棋子入場 numbness = True，**除了 ASS（刺客先攻）**。
- numbness 期間：攻擊指令完全無效（不消耗攻擊次數）、不能被選為移動對象。
- 清除時機**只有**：擁有者回合結束 settle 時（當次 on_settle 回傳 0 分）；
  以及少數卡牌效果（ADCB 的 token_draw 會用掉清除、被 APW/APR… 攻擊附加時重新變 True）。
- 對手回合不會清你的 numbness。

## 4. 行動規格（Dispatcher 的 action 集）

出處 `core/battling_dispatcher.py` `_execute()`。回合限定行動（attack/play_card/move_to/heal/spawn_cube/toggle_upgrade/end_turn）只有當前回合玩家可執行。

| action | 前置條件 | 效果 |
|---|---|---|
| `attack(x,y)` | `number_of_attacks[我]>0`；(x,y) 是我方棋子 | 執行 `card.attack()`；**成功才**扣 `attack_uses`（預設 1，HFBR=2），並記 HIT 統計 |
| `play_card(x,y,hand_index)` | 手牌存在 | `HEAL`→heals+1；`MOVE`→movings+1（進棄牌堆）；`MOVEO`→movings+1（直接消失）；`CUBES`→cubes+2；其餘為棋子：目標格有效且空 → 生成（Cyan `(+)` 前綴走升級價格檢查，錢不夠則生成失敗、牌留手上）→ deploy() 鉤子 → 佔格 |
| `move_to(x,y)` | 兩段式，見下 | |
| `heal(x,y)` | `number_of_heals>0`；(x,y) 我方棋子 | 回 6 HP；溢出部分 `//2` 轉護盾（armor）；Shadow 不可治療 |
| `spawn_cube(x,y)` | `number_of_cubes>0`；格子空 | 生成 neutral CUBE(4HP/0atk) |
| `toggle_upgrade(hand_index)` | 手牌以 `C` 結尾（Cyan 卡） | 名字加/去 `" (+)"` 後綴 |
| `end_turn` | — | 見 §3 |
| `quit` | — | 結束對戰 |

**移動（兩段式）**，出處 `player.py move_card()` + `cards/base.py move()`：

1. 沒有棋子處於 moving 狀態時：點擊我方**非 numbness** 棋子且 `number_of_movings>0`
   → 該棋子 `moving=True`、`number_of_movings -= 1`（**此時就扣**）。
2. 有棋子 moving 時：第一次點該棋子 → `mouse_selected=True`；再點目的地
   → `move()`：目的地必須是空格、且在 8 鄰格（切比雪夫距離 1）內。
   - 成功：釋放舊格、佔新格、發 move 事件、`after_movement()` 鉤子、
     全場棋子 `move_broadcast(mover)` 鉤子。
   - 失敗（不相鄰或格子被佔）：`moving=False`，**移動點數不退**。
   - `custom_move()` 鉤子可攔截整個移動（目前無卡使用，保留）。

## 5. 攻擊目標判定（attack patterns）

出處 `cards/base.py detection()` 與 `config/job_dictionary.json attack_type_tags`：

| pattern | 命中 | 使用職業 |
|---|---|---|
| `small_cross` | 上下左右 4 格 | TANK, LF |
| `large_cross` | 同行同列全部（不含自己） | ADC |
| `small_x` | 四斜角 4 格 | ASS |
| `small_cross small_x` | 九宮格 8 格（兩 pattern 聯集） | HF |
| `nearest` | 曼哈頓距離最近的**一個**敵人；並列時 `rng.choice` 隨機挑 1 | AP, APT |
| `farthest` | 最遠一個；並列隨機挑 1 | SP |
| `large_x` | **空實作**（不命中任何目標） | 無 |
| `None` | 不攻擊 | CUBE |

- 目標池 = 對手場上棋子 + **neutral 棋子**（CUBE、LUCKYBLOCK 會被打）。只挑 `health>0`。
- AOE pattern 命中多目標時逐一結算，每個目標間動畫延遲 `ANIM_LUNGE_STEP = 0.32s`。

## 6. 傷害管線（damage_calculate）

出處 `cards/base.py damage_calculate()`。**順序不可改**：

```
victim.damage_calculate(value, attacker, use_ability=True):
 0. victim.health <= 0 → 直接 return False
 1. attacker.hit_cards.append(victim)
 2. victim.damage_block(value, attacker)      # True → 整次攻擊無效（TANKC 升級版格擋）
 3. use_ability 時：attacker.ability(victim)   # 「攻擊附帶」類能力在這裡發動
       回 True → ABILITY 統計+1、ability_signal()
 4. value = attacker.damage_bonus(value, victim)   # 預設 value + extra_damage
 5. value = victim.damage_reduce(value)            # 減傷（APTC 升級版）
 6. value = victim.special_damage_interceptor(value, attacker)
       # 收集「全場所有棋子」的 on_field_effect_trigger(victim, value, attacker)
       # 每個回傳 (priority:int, new_value:int, feedback:Callable|None)
       # 依 priority 升冪逐一套用（後者覆蓋前者的 value），feedback 隨套用執行
       # 現況只有 APTF（優先度 20）使用
 7. 護盾結算：
    armor >= value        → armor -= value（本體無傷）
    0 < armor < value     → 溢出打血；若溢出 >= health 則 health=0
    armor == 0            → health -= value（value 超過 health 時記為實際扣血量）
    # 統計 DAMAGE_DEALT/DAMAGE_TAKEN 記「實際造成量」，log_attack
 8. victim.been_attacked(attacker, value) → been_attacked_signal()
 9. attacker.after_damage_calculated(victim, value)
10. health 歸 0 時：
    KILLED/DEATH 統計 → attacker.killed(victim) → killed_signal()
    → victim.been_killed(attacker) → been_killed_signal()
    → victim.can_be_killed() 為 True 才標記 pending_death + death 事件
      （HFR 怒氣、HFC 升級怒氣期間 can_be_killed=False → 屍體留場直到怒氣清除）
```

- 死亡移除不在管線內做：等 §3 的 logic_update 回收（`die()` 鉤子 → 棄牌堆）。
- 對 0 HP 但 can_be_killed=False 的棋子再攻擊：步驟 0 直接無效。
- 治療可以把 0 HP（尚未回收）的棋子救回來（例：怒氣中的 HFR）。

## 7. 攻擊遞迴防護（pending_attacks 佇列）

出處 `cards/base.py launch_attack()` / `enqueue_attack()`：

- 能力引發的「追加攻擊」（如 ADCB 藍球連鎖、Green 好運 case3）不得在傷害管線中同步執行，
  必須 `enqueue_attack()` 排入佇列。
- 最外層的 `launch_attack` 結束後負責 drain 佇列：逐一取出，攻擊者仍存活才執行；
  drain 中新增的請求會繼續處理。
- `SPB`（藍色水晶）deploy 結尾會 **清空** 佇列（防止入場傷害觸發連鎖）。
- Godot 實作：GameCore 內建 `_attack_draining` 旗標與 deque，語義照抄。

## 8. 治療規則

`Card.heal(value)`：`health+value <= max_health` → 直接加；超過 → 補滿，溢出量 `//2` 轉 armor。
Shadow 覆寫為不可治療。HEAL 魔法固定 +6。

## 9. 選秀 BP（本機模式）

出處 `core/draft_state.py` / `draft_dispatcher.py`：

- 階段：`p1_first6`（p1 選滿 ≥6）→ `p2_pick12`（p2 選滿 12）→ `p1_last6`（p1 補滿 12）→ done。
- 動作：add_card / remove_card（移除最後一張同名）/ remove_last_card / advance_phase /
  toggle_timer / toggle_file_save。
- 限制：牌組 12 張上限；單位同名 ≤2；魔法（CUBES/MOVE/HEAL）≤3。
- 展示館（exhibit）：依顏色分頁，每頁最多 8 職業 + 魔法列（CUBES/HEAL/MOVE）。
  頁面內容由 `card_setting.json` 有定義且已註冊的卡組成（Purple 只有 AP/TANK/HF/ASS）。

## 10. 動畫事件（CombatEvent）

出處 `shared/combat_event.py`。GameCore 對表現層的唯一輸出通道：

```
kind: "attack" | "hurt" | "float" | "move" | "death"
board_x, board_y          # 主角格
target_x, target_y        # attack 的方向 / move 的來源格
damage                    # float 用（飄字數值）
delay                     # 秒；排程用
post_health               # hurt 用（動畫結束時的血量顯示）
```

節奏常數：`ANIM_LUNGE_STEP = 0.32` 秒。多目標攻擊第 i 個目標：
attack 事件 delay = base + i×0.32；hurt/float 事件 delay = attack delay + 0.32×0.55。
`_attack_anim_cursor` 在一次行動內遞增，讓連續攻擊依序播放。
Godot 版事件種類可增（如 cast、spawn、token…），但至少涵蓋這五種與同樣節奏。

## 11. 統計（GameStatistics）

StatType（`shared/stat_type.py`）：`CARD_USE, HIT, DAMAGE_DEALT, DAMAGE_TAKEN, DAMAGE_TAKEN_COUNT,
SCORED, ABILITY, HEALING, HEAL_USE, MOVE, MOVE_USE, CUBE_USE, KILLED, DEATH, TOKEN_USE, ROUNDS_SURVIVED`
＋ `score_history`（每回合結束記一筆 score）。
key 格式：全域類用玩家名，卡牌類用 `owner_JOBANDCOLOR`（如 `player1_ASSW`）。
終局畫面用它畫長條/圓餅圖（`screens/end_game/`）。
