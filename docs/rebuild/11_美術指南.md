# 11 美術上手指南（Phase 14 產出）

> 給接手美術/視覺的人。**目標：不讀程式，就能在 Godot 編輯器裡把畫面改成你要的樣子。**
> 本文是 Phase 14「美術可編輯化」的成果總表；改造的盤查與逐項裁定見
> `08_場景編輯器化.md` §5（P14-1 盤查表），任務紀錄見 `06_任務清單.md` P14-1～P14-7。

## 0. 三十秒版

| 你想改什麼 | 去哪裡改 |
|---|---|
| 全案顏色、字級、按鈕樣式 | `theme/main_theme.tres`（見 §1） |
| 先手紅／後手藍／頁籤選中黃 | 同上，`Global` 型別的三個具名色（§1.2） |
| 某顆按鈕/卡片/列的長相 | 對應的 **item 模板場景**（§2） |

**開工前**：用 Godot **4.7 stable** 開啟專案根目錄。基準解析度 1024×768、
stretch 模式 `canvas_items`——**編輯器裡看到的就是遊戲裡的樣子**。

## 1. 全域樣式：Theme

### 1.1 色盤與字型

`theme/main_theme.tres` 掛在 `project.godot` 的 `gui/theme/custom`，
**所有 Control 自動繼承**（不必逐場景掛）。改這一份，全案生效。
完整色表與字型插槽說明見 **`theme/色盤說明.md`**。

在編輯器裡雙擊 `main_theme.tres` 會開啟 Theme 編輯面板，可視覺化調整
Label／Button／Panel 的字色、字級、底圖（StyleBox）、外框。

### 1.2 具名色（跨場景語意色）

| 具名色 | 意思 | 用在哪 |
|---|---|---|
| `Global/colors/player1_accent` | 先手 P1 的代表色 | 地格所有權外框、記分板、終局統計表與門檻線 |
| `Global/colors/player2_accent` | 後手 P2 的代表色 | 同上 |
| `Global/colors/tab_selected` | 頁籤/模式鈕「選中」染色 | battle 模式列、draft 色頁、圖鑑色頁與職業頁 |

這三個色**只定義在 theme 一處**，程式一律經 `UIPalette` 取用。改 theme 即全案生效，
不要在任何場景裡另外硬編一份。

### 1.3 派別色是資料，不是美術參數

十個派別（蒼白/緋紅/翠綠…）的顏色來自 **平衡資料** `data/balance/`，
程式一律走 `Balance.color_rgb`。**不要在場景或程式裡硬編派別色**——
它與 Python 版共用同一份數值來源，改動屬於平衡調整、不屬於美術。

## 2. Item 模板場景（P14-3）

清單裡「一顆一顆長得一樣」的東西（手牌鈕、房間列、卡片列…）都抽成了獨立場景。
**它們刻意不掛腳本**——你只會看到屬性，不會碰到程式。改一個檔案，全遊戲該類元件一起變。

| 場景 | 是什麼 |
|---|---|
| `scenes/battle/hand_card_button.tscn` | 手牌卡片鈕（可點） |
| `scenes/battle/hand_card_button_readonly.tscn` | 手牌卡片鈕（對手/旁觀，唯讀） |
| `scenes/draft/exhibit_card.tscn` | 選秀展示卡（含魔法卡） |
| `scenes/draft/deck_row_button.tscn` | 選秀牌組列 |
| `scenes/encyclopedia/card_row_button.tscn` | 圖鑑卡片列 |
| `scenes/encyclopedia/color_tab_button.tscn` | 圖鑑色系頁籤 |
| `scenes/encyclopedia/job_tab_button.tscn` | 圖鑑職業頁籤 |
| `scenes/encyclopedia/deriv_button.tscn` | 衍生物連結鈕 |
| `scenes/encyclopedia/attack_icon.tscn` | 攻擊範圍圖示 |
| `scenes/menu/replay_row_button.tscn` | 主選單回放列 |
| `scenes/online/room_row_button.tscn` | 大廳房間列 |
| `scenes/end_game/stat_cell.tscn` | 終局統計表儲存格／圖表文字 |
| `scenes/ui/keyword_tip.tscn` | 關鍵字懸停浮窗 |

字級、最小尺寸、對齊、底色都在這些場景裡；文字內容與點擊行為由程式填。
**要加圖示**：在 item 場景裡加 TextureRect/把 Button 的 `icon` 填上即可，程式不必改。

## 3. 棋盤幾何（P14-2）

打開 `scenes/battle/battle.tscn`：

- **`BoardAnchorOrtho`**（Node2D）＝俯視（方格）模式的棋盤**左上角**。用滑鼠拖它，
  格線、棋子、高亮、範圍預覽、投射物瞄準、點擊判定**整組跟著走**。
- **`BoardAnchorIso`**（Node2D）＝45 度（菱形）模式的棋盤**上頂角**，同樣可拖。
- 選 root 節點 `Battle`，Inspector 的 **「棋盤幾何」** 群組：
  `board_cell_size`（格內棋子邊長）、`board_ortho_stride`（俯視格距）、
  `board_iso_half_width`／`board_iso_half_height`（菱形半寬/半高）。

> ⚠️ `board_cell_size` 應與 `PieceView` 的 `CELL_SIZE`（96）一致——它只影響
> 「棋子左上角相對格心的偏移」，不改變棋子畫多大；兩者不一致棋子會偏離格心。

同一顆 `Battle` 節點上還有 **「棋盤高亮配色」** 群組（懸停/範圍/選取/移動/AI 目標圈/待放置染色）。

## 4. 素材：丟檔案即生效（P14-5）

資產根一律是 **`res://img/`**。查檔規則相同：**有檔就用、沒檔就退回現在的幾何/純色外觀**，
所以你可以一次只放一張圖慢慢換，中途畫面不會壞掉。

| 放什麼 | 放哪裡 | 沒放的話 |
|---|---|---|
| 棋子圖 | `img/piece/card/<card_id>.png` | 現行幾何佔位形（多邊形＋職業碼） |
| 棋盤底圖（俯視） | `img/board/skin_ortho.png` | 只有格線 |
| 棋盤底圖（45 度） | `img/board/skin_iso.png` | 只有格線 |
| 場景背景 | `img/UI/bg/<場景名>.png` | `.tscn` 裡的純色 `Background` |

- **場景名**＝`main_menu`／`draft`／`battle`／`end_game`／`encyclopedia`／`online_lobby`。
- **card_id** 可在遊戲內「百科圖鑑」或 `data/balance/card_setting.json` 查到（如 `ADCW`、`TANKBR`）。
- 棋子圖預設**等比縮放到剛好塞滿一格**（96px）。想照原尺寸：取消 `piece_view.tscn` 根節點
  「美術素材 → `sprite_fit_cell`」。想換目錄：改同群組的 `sprite_dir`。
- 鏡像棋子（SHADOW）沿用**本體職業**的圖，自動半透明。
- 棋盤底圖的對齊點：俯視圖**左上角**對 `BoardAnchorOrtho`、45 度圖**上邊中點**對 `BoardAnchorIso`。
- 場景背景圖預設「保持長寬比填滿」；要改拉伸/平鋪，選該場景的 `BackgroundImage` 改 Stretch Mode。

各目錄裡都有一份 `README.md` 寫著同樣的規則，改圖時就近可查。

**狀態圖示**（麻痺/憤怒/移動）已經是 `piece_view.tscn` 裡的三顆 TextureRect
（`NumbnessIcon`／`AngerIcon`／`MovingIcon`），直接換 `texture` 屬性即可。

## 5. 特效與動畫參數（P14-6）

所有演出參數都是 Inspector 上的欄位，**預設值＝目前畫面的樣子**，放心調。

| 調什麼 | 在哪個場景的哪個節點 |
|---|---|
| 受擊白閃、抖動、命中頓幀 | `piece_view.tscn` root →「特效：受擊」 |
| 受擊火花、死亡碎片（顆數/速度/大小/色） | 同上 →「特效：粒子」 |
| 死亡淡出、旋轉、殘影 | 同上 →「特效：死亡與殘影」 |
| 施法脈動與能量環 | 同上 →「特效：施法環」 |
| 拉弓後拉、近戰撲擊距離、移動滑行 | 同上 →「特效：攻擊位移」 |
| 棋子佔位形的描邊/CUBE/幸運方塊填色 | 同上 →「佔位美術配色」 |
| 箭矢顏色與形狀 | `scenes/battle/projectile.tscn` |
| 命中閃光顏色、半徑、時長 | `scenes/battle/impact_flash.tscn` |
| 傷害飄字（字級/色/上飄距/時長） | `anim_demo.tscn` 的 `CombatScheduler` 節點 |
| 擊殺鏡頭震動、資源獲得飄字 | `battle.tscn` root →「特效：鏡頭震動」「特效：資源飄字」 |
| 每張卡的攻擊節奏與特效色 | `PieceAnimationSet` 資源（`lunge_step`／`hit_ratio`／`fx_color`／投射物/命中場景） |

> 遊戲內按 **I** 可切「瞬時模式」（動畫全關）。**瞬時模式下應該完全沒有特效**——
> 這是規則層要求（動畫不得影響對局結果），有測試守著，調參數不會破壞它。

## 6. 兩個預覽場景（按 F6 執行）

- **`scenes/battle/piece_gallery.tscn`** — 全部棋子一覽（10 色 × 8 職業＋特殊卡＋鏡像）。
  副標題顯示「貼圖 n / 全部 m」，一眼看出哪些卡還沒圖；按 **S** 在「美術貼圖／幾何佔位」

  **空白鍵**重播、**I** 切瞬時模式。調完 §5 的參數回這裡按 F6 就能看效果。

## 7. 禁區（改了會壞，或屬於程式/平衡）

1. **`%` 場景唯一名稱不可改名、不可刪節點**（Inspector 裡節點名旁有引號圖示的那些）。
   程式以這些名稱抓節點，改名＝當場崩。**搬位置、改樣式、改屬性都沒問題**；
   要改名或刪節點請開程式任務。
2. **容器結構不可重組**：`LeftHandBox`／`RightHandBox`／`CardGrid` 等會被程式塞入子節點，
   可以搬移、改大小、換樣式，但不要把它們換成別的型別或塞進別的父節點。
3. **`script/core` 一切不可動**：那是規則核心，與 Python 版對齊，動了就改變對局結果。
4. **派別色不可硬編**（見 §1.3）；平衡數值（`data/balance/`）不是美術資產。
5. **`online_lobby.tscn` 的 `NetClient` 不可搬移**：連線的 RPC 依賴它的節點路徑。
6. **手牌左右欄的歸屬不可對調**：那是已定案的規則（D21，主視角恆左欄）。
7. 改完請跑一次測試（見下），確認沒動到行為。

## 8. 改完怎麼確認沒弄壞

在專案根目錄執行：

```
& "d:\Code\Games\Godot_v4.7-stable_win64.exe" --headless --path "<專案根>" -s tests/run_tests.gd
```

看到 `NNNN passed`（沒有 `failed`）就是好的。若動到 `.tscn` 的節點名稱或結構，
測試會直接指出哪個節點找不到。
