# 02 卡牌能力總表（Python 4.3.0.0 為準）

> 數值一律以 `config/card_setting.json`（canonical，將同步到 Godot `data/balance/`）為準，
> 本文寫的數字只是快照方便閱讀。能力描述以 **Python 程式行為** 為準（出處：`cards/card_<color>.py`）。
> 「鉤子」欄對應 `04_架構設計.md` §5 的觸發點命名。

## Green 幸運（運氣值機制）

**lucky_effects(target)**（共用函式）：擲 1–100 ≤ target 擁有者運氣 → **好運**：運氣+1，隨機一項
{護盾+4、ATK×2、立即排入一次攻擊、獲得移動、四斜角空格生 LUCKYBLOCK}；
否則 **壞運**：運氣-1，隨機一項{護盾歸 0、麻痺、HP 減半、ATK 減半、HP-2（HP≥2 才扣）}。
變體旗標：`AP`（自己：壞運分支直接跳過、好運不生方塊）、`AP_target`（**必定走壞運分支**，見規則 B2）、
`TANK`（好運分支直接跳過＝只可能壞運或沒事）。

## Fuchsia 紫紅（鏡像 Shadow）

**Shadow 通則**：HP1/ATK0、佈署在本體的中心對稱位（`(3-x, 3-y)`）、不佔攻擊能力
（攻擊時目標交給 linker 用 linker 的 ATK 打）、不能被治療、0 分、本體移動時鏡像同步移動（movable 的 shadow）。
Shadow 死亡時若格上無其他棋子則釋放格子。**Shadow 斬殺目標時**：若 linker 是 ASSF → 在受害者位置再生一個 shadow。
Shadow 承傷時若 linker 是 APTF → linker 獲得 value//2 護盾。

| APF | 3/1 | 佈署生鏡像；佈署時與每回合開始：站在鏡像格上的敵人麻痺（光環）；攻擊附帶麻痺 |
