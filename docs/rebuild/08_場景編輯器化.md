## 1. 去 v2 命名（P7-1 / P7-2）

- **目錄**：`script_v2/` → `script/`、`scenes_v2/` → `scenes/`（git mv 保留歷史）。
  全專案 `res://script_v2|scenes_v2` 路徑替換（.gd / .tscn / project.godot）。
- **類別**：所有 `class_name XxxV2` → `Xxx`（如 `GameEventV2→GameEvent`、`CombatV2→Combat`、
  `TriggerV2→Trigger`、`AbilityV2→Ability`、`PieceViewV2→PieceView`…共 29 個）。
  舊碼已刪，名稱無衝突。統一規則：識別字 `V2` 一律移除（`sed -E 's/([A-Za-z_])V2\b/\1/'`）。
- **註解**：`v2` 字眼移除或改寫（如「能力系統 v2」→「能力系統」；game_event.gd 的
  「避開舊碼 GameEvent 衝突」註解已無意義，改寫）。
- 歷史文件（進度日誌、06 已勾任務的備註）**保留原文**——它們是紀錄，不改寫歷史。
- 驗收：`--import` 無錯；全測試套件綠；launch 煙霧（main_menu 正常啟動）。

## 2. 場景編輯器化準則（P7-3 ~ P7-6 共同規範）

**目標**：打開任一 `.tscn`，編輯器裡就能看到完整 UI 骨架；美術可直接改節點、換圖，不讀程式。

1. **靜態節點進 `.tscn`**：背景、標題、Label、Button、Panel、容器（HBox/VBox/Grid）、
   圖層 Node2D（BoardLayer/FxLayer…）、棋盤格線——一律在場景檔宣告。
2. **動態集合仍由程式生成**，但必須生成到 `.tscn` 內宣告好的**容器節點**下：
   手牌鈕 → `HandBox`；展示館卡 → `ExhibitGrid`；棋子視圖 → `BoardLayer`；
   飄字/投射物 → `FxLayer`；圖表折線/長條 → `ChartLayer`/`BarsBox`。
3. **場景唯一名稱（Scene Unique Names）**：程式要取用的節點在編輯器打開
   `unique_name_in_owner`（`%` 前綴），腳本一律 `%NodeName` 取節點——
   美術搬動節點位置/層級不會弄壞程式。
4. 腳本用 `@onready var _x := %NodeName` 取代 `_ensure_built()` 程序建構；
   移除程序建構路徑（不留雙軌）。`configure()`/行動方法等對外 API 不變。
5. 佈局：HUD 控件掛 `CanvasLayer` 下、位置/字級/顏色由 `.tscn` 屬性定義（theme override 進場景檔）；
   Node2D 世界（棋盤、棋子）維持座標制。
6. **測試調整**：headless 測試由 `Script.new()` 改為 `load("....tscn").instantiate()`；
   節點取得走場景樹（`%` 名稱）。每個場景任務附「節點樹存在」斷言（instantiate 後檢查關鍵節點）。
7. 規則不動：場景腳本仍是薄殼（輸入→`GameCore.dispatch`、事件→動畫、狀態→UI），core 不受影響。

## 3. 各場景目標節點樹（宣告進 .tscn 的部分）

### piece_view.tscn（P7-3）
```
PieceView (Node2D) + piece_view.gd
├─ VisualRoot (Node2D)
│   ├─ OutlineShape (Polygon2D)
│   ├─ PlaceholderShape (Polygon2D)
│   └─ SpriteSlot (Sprite2D, visible=false)   # 美術插槽
├─ StatsOverlay (Node2D)
│   ├─ JobLabel / NameLabel / HealthLabel / AttackLabel / ArmorLabel / ExtraLabel (Label)
└─ StatusIcons (Node2D)
    ├─ numbness / anger / moving (TextureRect，貼好 buff 圖)
```
呼叫端（battle/gallery/anim_demo/tests）改為 instantiate `piece_view.tscn`。

### battle.tscn（P7-4）
```
Battle (Node2D) + battle.gd
├─ Background (ColorRect)
├─ GridLayer (Node2D：16 格線 Line2D 預置)
├─ PersistLayer / PreviewLayer (Node2D：高亮/預覽，_draw 由腳本觸發)
├─ BoardLayer (Node2D)      # 棋子視圖容器（動態）
├─ FxLayer (Node2D)         # 投射物/飄字容器（動態）
└─ HUD (CanvasLayer)
    ├─ ScoreLabel / TurnLabel / ResLabel / CountsLabel / HintLabel (Label)
    ├─ ModeBar（攻擊/移動/治療/方塊 Button）
    ├─ UpgradeBtn / HintToggle / AnimToggle / EndTurnBtn (Button)
    ├─ HandBox (HBoxContainer)   # 手牌鈕容器（動態）
    └─ WinPanel (Panel)：WinLabel + 再來一局/終局統計/回主選單
```

### draft.tscn（P7-5）
```
Draft (Node2D) + draft.gd
├─ Background (ColorRect)
└─ HUD (CanvasLayer)
    ├─ TitleLabel / PhaseLabel / MsgLabel / HelpLabel (Label)
    ├─ ColorTabs (HBoxContainer：10 色鈕預置)
    ├─ ExhibitGrid (GridContainer)   # 展示卡（動態）
    ├─ MagicBox (HBoxContainer)      # 魔法卡（動態）
    ├─ P1DeckPanel / P2DeckPanel (VBoxContainer＋標題)   # 牌組列（動態）
    └─ AdvanceBtn / RemoveLastBtn / TimerBtn / FileBtn (Button)
```

### main_menu.tscn / end_game.tscn（P7-6）
```
MainMenu：Background + HUD(Title/Subtitle/選單五鈕/MsgLabel/VersionLabel/SettingsPanel[兩開關+返回])
EndGame ：Background + ChartFrame(ColorRect) + ChartLayer(Node2D，折線動態)
          + HUD(TitleLabel/ChartCaption/BarsRoot[長條動態]/再選秀鈕/回選單鈕)
```
end_game 由 battle `configure()` 帶資料，圖表內容動態、框架宣告。

## 4. 驗收方式

- P7-1/P7-2：機械改名——全套件綠 + launch 煙霧即可勾選。
- P7-3 ~ P7-6：行為保持重構——(a) instantiate 場景後關鍵節點存在（`%` 名稱解析成功）；
  (b) 既有 headless 測試（改 instantiate 後）全綠；(c) in-tree 煙霧無誤。滿足即可勾選。
- P7-7：人工在編輯器打開各 `.tscn` 確認「看得到、可編輯」，跑全流程確認外觀/行為不變，
  依既有驗收表抽查。人工確認後 Phase 7 收官。
