# 全域 Theme 色盤說明（P8-2）

`theme/main_theme.tres` 是全專案的預設 Theme，透過 `project.godot` 的
`gui/theme/custom` 掛成 `ThemeDB.project_theme`——**所有 Control 自動繼承**
（不受 Node2D / CanvasLayer 祖先影響）。四場景 root 是 Node2D（無 theme 欄位），
故採「專案預設主題」而非逐 root 掛載，效果等同且更全面。

## 具名色（accent，P14-4 起）

跨場景共用的語意色已收斂進 theme 的自訂型別 `Global`，**單一來源、改一處全案生效**：

| 具名色 | 色值 | 用在哪 |
|---|---|---|
| `Global/colors/player1_accent` | `0.95, 0.4, 0.4` | 先手 P1：地格所有權外框、記分板、終局統計表與門檻線 |
| `Global/colors/player2_accent` | `0.45, 0.6, 1` | 後手 P2：同上 |
| `Global/colors/tab_selected` | `1, 1, 0.6` | 頁籤/模式鈕的「目前選中」染色（battle 模式列、draft 色頁、圖鑑色頁與職業頁） |

**取用方式**：一律走 `UIPalette`（`script/view/ui_palette.gd`）——
`UIPalette.player_color("player1")`／`UIPalette.tab_tint(是否選中)`。
不要直接 `get_theme_color()`：對戰/選秀/圖鑑的場景 root 是 **Node2D**（沒有 theme 查詢 API），
`UIPalette` 改查 `ThemeDB.get_project_theme()`，兩者拿到的是同一份 theme。

改色只要編輯 `main_theme.tres` 的這三行（或在編輯器的 Theme 面板改 `Global` 型別的顏色）。

> P14-4 前這三個語意色在 battle／scoreboard／end_game／draft／encyclopedia 各自硬編一份
> （共 11 處），改一個顏色要翻五個檔。`tests/test_no_hardcoded_colors.gd` 現在會擋住
> 「在場景腳本邏輯裡直接寫 `Color(...)`」的回頭路（`@export` 宣告處不受限，那正是參數該待的地方）。

**單一場景自用的顏色**（棋盤高亮、圖表配色、記分板中線、佔位美術色…）不進 theme，
改用該場景腳本的 `@export`——在編輯器選中場景 root 就能調，見各檔的 `@export_group`。

## 派別色（不在此 theme 內）

10 色派別色是**資料驅動**，一律走 `Balance.color_rgb(card_id)`（來源 `data/balance`，
見 05）。棋子/卡片/圖鑑用它上色，**不要**在 theme 或程式硬編派別色。

## 字型插槽（之後換字型只改這裡）

目前用引擎內建字型（未設 `default_font`）。要換自訂字型：
把字型檔（.ttf/.otf → FontFile）放入專案，在 `main_theme.tres` 的 `[resource]` 段加：
```
default_font = ExtResource("<字型檔>")
```
並在檔頭 `[ext_resource ...]` 宣告該字型。全專案文字即一次換掉，無需動任何場景。

## 字級基準

- `default_font_size = 16`（未特別指定者）
- Button 預設 15、Label 預設 15；標題類（44/34/26/22/20…）以逐節點 `font_size` override。
